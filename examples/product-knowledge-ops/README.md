# Product Knowledge Ops Example

This anonymized example shows how one product knowledge operations portfolio can be represented as an
LLM-compiled knowledge base.

Projects:

- Core Knowledge Portal
- Flexible Review Workflow
- Knowledge AI Assistant
- Team Workflow Expansion

The example demonstrates:

- one raw source affecting multiple projects;
- one project depending on another;
- multiple projects sharing a capability;
- `current_draft/` and `current/` separation;
- source cards and source manifest;
- a mini-kb for review preparation.

No real company names, users, or sensitive business details are used.
